## Usage

`/migrate $ARGUMENTS`

- `/migrate <name>` — create a new migration with the given name and regenerate the client
- `/migrate deploy` — apply pending migrations and regenerate the client
- `/migrate generate` — regenerate the client only (no migration changes)

## Context

- Schema files live in `prisma/schema/` (modular setup via `prisma.config.ts`)
- Generated client outputs to `prisma/src/generated/prisma/` — not the standard location
- PostgreSQL must be running (port 5433 locally, or inside Docker)

## Steps

**New migration (`/migrate <name>`):**
```bash
npx prisma migrate dev --name <name>
npx prisma generate
```

**Deploy existing migrations (`/migrate deploy`):**
```bash
npx prisma migrate deploy
npx prisma generate
```

**Regenerate client only (`/migrate generate`):**
```bash
npx prisma generate
```

After running, confirm the client was regenerated by checking that `prisma/src/generated/prisma/` was updated.
